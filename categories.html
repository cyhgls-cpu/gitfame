项目分类 ​

分类体系 ​

我们按照以下分类体系组织项目，确保每个项目都有明确的归属和收录标准。

1. 智能前沿 (AI & ML) ​

二级分类：

  • LLM 框架
  • AI Agent
  • 本地大模型 (Ollama/Local)
  • 图像/视频生成

创新细节/收录侧重：

  • 侧重于"一键运行"和"低门槛接入"的项目
  • 关注模型性能和易用性
  • 优先收录有实际应用场景的项目

2. 开发利器 (DevTools) ​

二级分类：

  • 终端增强 (CLI)
  • API 工具
  • 高效 IDE 插件
  • 测试/调试

创新细节/收录侧重：

  • 标注"提升效率百分比"，强调对工作流的改造
  • 关注工具的稳定性和可靠性
  • 优先收录能够解决实际开发痛点的项目

3. 现代 Web (Web Stack) ​

二级分类：

  • 全栈框架
  • UI 组件库
  • 低代码 (Low-code)
  • 动效/可视化

创新细节/收录侧重：

  • 增加"性能评分"和"设计风格"标签（如：Bento, Minimal）
  • 关注用户体验和开发效率
  • 优先收录有创新性设计的项目

4. 基础设施 (Infra) ​

二级分类：

  • Docker/K8s 工具
  • 数据库/存储
  • 安全/防护
  • 网络/穿透

创新细节/收录侧重：

  • 侧重于"高可用"和"轻量化"替代方案
  • 关注系统的稳定性和安全性
  • 优先收录能够简化基础设施管理的项目

5. 自托管 (Self-Hosted) ​

二级分类：

  • 云服务替代品
  • 家庭实验室 (NAS)
  • 私人影音
  • 自动化流

创新细节/收录侧重：

  • 核心卖点：隐私保护、数据自主、部署成本
  • 关注项目的可维护性和扩展性
  • 优先收录适合个人和小团队使用的项目

6. 终身学习 (Resources) ​

二级分类：

  • 技术路线图 (Roadmap)
  • 电子书合集
  • 面试/算法
  • 优质论文

创新细节/收录侧重：

  • 剔除单纯的链接堆砌，只收录"系统性"的项目
  • 关注内容的质量和实用性
  • 优先收录有结构化内容的项目

GitFame 分类 ​

  • 一级分类：开发利器 (DevTools)
  • 二级分类：终端增强 (CLI)
  • 收录侧重：提升效率百分比，强调对工作流的改造
  • 效率提升：相比手动分析，提升效率达 95% 以上
